processlist_repl

NAME

processlist_repl: Listing of replication processes: the server's slave threads and any replicating slaves

TYPE

View

DESCRIPTION

processlist_repl displays only replication processes. These may include:

  • Connected slaves: when this server acts as a master, each slave connects using a single process. A slave with running IO thread will appear in this server's processlist_repl
  • IO thread: if this server is itself a replicating slave
  • SQL thread: if this server is itself a replicating slave

This view provides with a quick look at replication status processlist-wise.

The output of this view is a row subset of INFORMATION_SCHEMA.PROCESSLIST.

STRUCTURE

mysql> DESC common_schema.processlist_repl;
+---------+-------------+------+-----+---------+-------+
| Field   | Type        | Null | Key | Default | Extra |
+---------+-------------+------+-----+---------+-------+
| ID      | bigint(4)   | NO   |     | 0       |       |
| USER    | varchar(16) | NO   |     |         |       |
| HOST    | varchar(64) | NO   |     |         |       |
| DB      | varchar(64) | YES  |     | NULL    |       |
| COMMAND | varchar(16) | NO   |     |         |       |
| TIME    | int(7)      | NO   |     | 0       |       |
| STATE   | varchar(64) | YES  |     | NULL    |       |
| INFO    | longtext    | YES  |     | NULL    |       |
+---------+-------------+------+-----+---------+-------+

SYNOPSIS

Structure of this view is identical to that of INFORMATION_SCHEMA.PROCESSLIST table

On Percona Server, this additional info is included:

  • TIME_MS: execution time in milliseconds

EXAMPLES

Show all replication processes

mysql> SELECT * FROM common_schema.processlist_repl;
+----------+-------------+-------------+------+-------------+---------+------------------------------------------------------------------+------+------------+
| ID       | USER        | HOST        | DB   | COMMAND     | TIME    | STATE                                                            | INFO | TIME_MS    |
+----------+-------------+-------------+------+-------------+---------+------------------------------------------------------------------+------+------------+
| 31066726 | repl_user   | sql02:48924 | NULL | Binlog Dump | 1205439 | Has sent all binlog to slave; waiting for binlog to be updated   | NULL | 1205438922 |
|  3598469 | repl_user   | sql01:51157 | NULL | Binlog Dump | 4445559 | Has sent all binlog to slave; waiting for binlog to be updated   | NULL | 4445558495 |
|  3598335 | system user |             | NULL | Connect     |  359428 | Has read all relay log; waiting for the slave I/O thread to upda | NULL |          3 |
|  3598334 | system user |             | NULL | Connect     | 4445564 | Waiting for master to send event                                 | NULL | 4445563890 |
+----------+-------------+-------------+------+-------------+---------+------------------------------------------------------------------+------+------------+

In the above example we see two slaves replicating from this server (sql02 & sql01), and the two threads (IO thread, SQL thread) this server uses to replication from its master

ENVIRONMENT

MySQL 5.1 or newer. Percona Server yields a different schema.
